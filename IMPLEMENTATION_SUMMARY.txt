🎯 HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

TWO-PART MATCHING ALGORITHM:

User Resume (PDF)
    ↓
Extract skills: ["arrays", "sorting", "binary-search"]
    ↓
    ├─── SEMANTIC MATCHING (70% weight) ────→ SBERT embedding similarity
    │                                          Result: 0.480
    │
    └─── TAG MATCHING (30% weight) ─────────→ Direct skill-to-tag match
                                               Result: 0.667
                                               (2 of 3 tags matched)
    ↓
Combined Score = (0.480 × 0.70) + (0.667 × 0.30) = 0.536
    ↓
Results ranked by combined_score (highest first)

🔑 KEY CONCEPTS
═══════════════════════════════════════════════════════════════════════════════

Tag:               Granular technical concept (e.g., "binary-search")
Tag Overlap:       Percentage of question tags matching user skills
Semantic Score:    SBERT embedding similarity (70% weight)
Tag Score:         Skill-to-tag match (30% weight)
Combined Score:    Final relevance ranking used for results
Use Tag Boosting:  Enable/disable tag-based matching in search

❓ COMMON QUESTIONS
═══════════════════════════════════════════════════════════════════════════════

Q: Do I need to use tags?
A: No! System works without tags. Tags improve relevance by ~17%.

Q: How long does implementation take?
A: Minimum 30 min (just add tags + rebuild)
   Full setup: 2-2.5 hours (all optional phases)

Q: Can I start small?
A: Yes! Add tags to 10 questions, rebuild, test, then scale up.

Q: What if I don't have tags yet?
A: System uses topic as fallback tag. Add tags when you have them.

Q: How do I choose good tags?
A: See SAMPLE_QUESTIONS_WITH_TAGS.csv & TAG_MATCHING_GUIDE.md

Q: Can I adjust the 70/30 split?
A: Yes! Change tag_weight parameter (default 0.3)

Q: Will it slow down search?
A: Minimal overhead (~5ms per 30 results)
